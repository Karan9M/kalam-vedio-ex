# UPSC Hinglish Script Rubric

## Purpose
This rubric defines how narration should be written before TTS is generated so the spoken result sounds exam-grade, natural, and visually alignable.

## Core Rules
1. Speak ideas, not symbols.
2. One spoken thought should map to one visual beat.
3. Keep sentences short enough to animate clearly.
4. Prefer meaning-first phrasing over notation-first phrasing.
5. Every scene should end with one exam takeaway.

## Do
- Say: "तेतीस दशमलव तैंतीस प्रतिशत यानी लगभग एक-तिहाई"
- Say: "plus 10 percent और minus 10 percent cancel नहीं होते"
- Say: "अगर B सौ है, तो A एक सौ पच्चीस होगा"
- Convert formulas into spoken explanations before showing the compact symbolic form
- Use Hinglish only where it sounds natural to aspirants

## Avoid
- Slash-heavy speech like "one by eight by thirty three by thirty three"
- Symbol dumps like "5 US dollars backslash" or raw punctuation narration
- Two unrelated concepts in the same sentence
- Decorative filler that does not advance the explanation

## Scene Writing Template
- Hook: what is the concept/problem?
- Decode: what should the student notice?
- Visual proof: what example/formula/comparison demonstrates it?
- Takeaway: what should the aspirant remember in the exam?

## Output Standard
For each scene, prepare:
- `title`
- `narration segments`
- `cue mapping`
- `takeaway`

This lets TTS, visuals, and timing all align against the same authored structure.
